# 第二节 传统方式实现搜索

# 1、SQL 语句

传统搜索方式其实就是直接到数据库里找，那当然需要编写 SQL 语句。在 SQL 语句中只能通过在 WHERE 子句中使用 LIKE 条件进行模糊查询。

select ... from ... where xxx_xxx like "%keyword%"
1

# 2、问题1：速度慢

WHERE 子句中 LIKE 条件是逐条检查数据库表中的每一条记录，把符合条件的返回。如果表中数据量很大（几百万条记录），搜索时间会非常长。

# 4、问题3：不支持相关性评分

对待用户的搜索，并不是把关键词匹配的所有记录都展示出来就完了，而是要尽可能的把用户需要的数据靠前显示。

举个具体例子来说：用户搜索『咖啡厅』，背后的真实意图会是什么呢？

  • 可能性A：为了开咖啡厅选址或找工商管理部门注册。
  • 可能性B：找一家咖啡厅喝咖啡。

我们都知道上面两种情况都有可能，但是很显然第二种情况概率大得多。所以一个完善的搜索引擎肯定需要根据各方面的因素来尽可能接近用户意图；并对数据样本进行打分，为用户返回和关键词相关性更强的搜索结果。在现在信息爆炸的年代，我们更追求数据的精准而不是大量呈现。

很显然 SQL 没法计算相关性的。
