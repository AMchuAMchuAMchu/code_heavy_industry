# JVM 笔记

# 1、总体机制

JVM 总体工作机制中比较重要的流程：

  • 类加载器子系统加载类到内存中
  • 类相关数据具体是保存到方法区中
  • 类创建对象，对象保存到堆空间
  • 通过类或对象调用方法，方法每执行一次，产生一个栈帧
  • 栈帧保存到 Java 栈中，顺序是先进后出，后进先出
  • 具体的字节码指令会被执行引擎翻译成 CPU 能够执行的指令

# 2、类加载器

# ①分类

  • 启动类加载器：加载核心包 rt.jar 中的类
  • 扩展类加载器：加载扩展 jar 包中或其他目录下的类
  • 应用类加载器：加载程序员开发的类和第三方 jar 包中的类

# ②获取方式

  • 基本方式：clazz.getClassLoader() 方法
  • 注意：启动类加载器是 C++ 写的，Java 环境中只能获取到 null 值

# ③双亲委派机制

  • 第一种情况：当前要加载的类属于应用类加载器负责的范围
    • 第一步：应用类加载器将任务交给扩展类加载器
    • 第二步：扩展类加载器将任务交给启动类加载器
    • 第三步：启动类加载器在核心范围查找这个类
      • 能找到：读取字节码文件，整个加载操作完成
      • 没找到：
        • 将任务还给扩展类加载器
        • 扩展类加载器在扩展范围查找这个类
          • 能找到：读取字节码文件，整个加载操作完成
          • 没找到：将任务还给应用类加载器
            • 应用类加载器在自己负责的范围查找
              • 能找到：读取字节码文件，整个加载操作完成
              • 没找到：抛出 ClassNotFoundException
  • 第二种情况：当前要加载的类属于扩展类加载器负责的范围
    • 第一步：扩展类加载器将任务交给启动类加载器
    • 第二步：启动类加载器在核心范围查找这个类
      • 能找到：读取字节码文件，整个加载操作完成
      • 没找到：
        • 将任务还给扩展类加载器
        • 扩展类加载器在扩展范围查找这个类
          • 能找到：读取字节码文件，整个加载操作完成
          • 没找到：宣告查找失败，整个加载操作完成
  • 第三种情况：当前要加载的类属于启动类加载器负责的范围
    • 启动类加载器在核心范围查找这个类
      • 能找到：读取字节码文件，整个加载操作完成
      • 没找到：宣告查找失败，整个加载操作完成

# 3、方法区

# ①存储内容

  • 类和接口本身的数据
    • 构造器
    • 方法
  • static 修饰的成员
    • 类变量
    • 常量
  • 常量池

# ②概念

  • 标准层面：方法区
  • 具体实现层面：
    • JDK 版本 ≤ 1.7 时：永久代
    • JDK 版本 ≥ 1.8 时：元空间

# ③永久代

  • 标准层面：属于堆空间
  • 具体实现层面：不属于堆空间

# 4、Java 栈

# ①栈帧

# [1]概念

方法每一次执行时所需要保存的相关数据保存到一个栈帧中。

方法每执行一次，生成一个栈帧。

# [2]栈帧中存储的数据

和我们平时编写的代码相关的数据：

  • 方法形参变量
  • 方法体内声明的变量
  • 方法返回值变量

# [3]栈帧结构

  • 局部变量表：保存方法中声明的所有变量
  • 操作数栈：方法运行过程中的临时数据
  • 帧数据区：保存访问常量池指针，异常处理表

# [4]父子关系

栈帧通过父子关系可以确定方法的调用关系：

  • 父帧：指向当前栈帧的调用者
  • 子帧：当前栈帧调用的下一个栈帧

# ②栈溢出错误

产生原因：Java 栈给当前线程分配的空间被占满后，还要申请更多空间。

全类名：java.lang.StackOverflowError

# ③Java 栈线程私有

某一个线程抛出 StackOverflowError 不会影响其他线程，Java 栈会给每个线程单独分配空间。

# 5、堆空间

# ①总体结构

# ②工作流程

  • 新创建的对象：进入 Eden 区。
  • Eden 区的对象经过 GC 后没有被释放：进入 Survivor 区。
  • Survivor 区分成 from 区和 to 区两个部分。
  • Survivor 区在执行 GC 时：
    • 将不释放的对象从 from 区复制到 to 区。
    • 将 from 区整个标记为空闲。
    • 交换 from 和 to 这两个指针。
    • 口诀：复制必交换，谁空谁为 to。
  • 新生代中的对象默认经历 15 次 GC 后，如果还幸存，则直接进入老年代。
  • 如果幸存者区已经满了，即使某个对象尚不到 15 岁，仍然会被移动到老年代

# 7、生产实践

# ①堆内存空间大小设置的建议。

  • 最大堆内存和初始化堆内存设置为一致：直接一步到位设置为能够使用的最大值，免除频繁申请空间的操作。

    • 空余堆内存小于40%时，JVM就会增大堆直到-Xmx的最大限制
    • 空余堆内存大于70%时，JVM会减少堆直到 -Xms的最小限制

  • 新生代大小：Sun官方推荐配置为整个堆的3/8

# ②Java 栈每个线程占用空间大小如何设置？

JDK5.0 以后每个线程堆栈大小为 1M，以前每个线程堆栈大小为 256K。根据应用的线程所需内存大小进行调整。在相同物理内存下，减小这个值能生成更多的线程。但是操作系统对一个进程内的线程数还是有限制的，不能无限生成，经验值在3000~5000左右。一般小的应用， 如果栈不是很深， 应该是128k够用的。大的应用建议使用256k。这个选项对性能影响比较大，需要严格的测试。
